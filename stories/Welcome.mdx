# 🏗️ GB Locações — Storybook Design System

Bem-vindo ao sistema de design da **GB Locações**, uma plataforma especializada
em locação de equipamentos para construção civil.

## 🎯 O que você encontrará aqui

Este Storybook reproduz **fielmente** as seções reais do site GB Locações,
preservando:

### ✅ Fidelidade Visual 100%

- **Cores exatas**: `#ea580c` (primária), `#3b82f6` (secundária)
- **Typography real**: Inter (corpo) + Jost (títulos)
- **Spacing idêntico**: Tokens de design preservados
- **Animações originais**: Framer Motion e CSS animations

### 📱 Responsividade Completa

- **Mobile First**: Breakpoints exatos do app
- **Tablets**: Layout intermediário
- **Desktop**: Experiência completa

### 🧩 Componentes Reais

- Reutilização 100% dos componentes do projeto
- Props e estados idênticos ao app
- Dados tipados compatíveis com Prisma

## 🗂️ Estrutura de Navegação

### 📄 Páginas Completas

- **Homepage** — Página inicial completa integrada

### 🎨 Seções Individuais

- **Hero** — Seção principal com CTA
- **Categories** — Grid de categorias de equipamentos
- **Featured** — Equipamentos em destaque
- **Why Choose Us** — Benefícios da empresa
- **Contact** — Formulário e informações de contato
- **Header** — Navegação global

### 🔧 Sistema de Design

- **Design Tokens** — Cores, typography, spacing
- **Componentes Base** — Button, Card, Input, etc.
- **Diretrizes** — Regras de uso e antipadrões

## 🎪 Recursos Especiais

### 🎛️ Controles Interativos

- **Reduced Motion**: Toggle para acessibilidade
- **Viewports**: Teste em diferentes dispositivos
- **Backgrounds**: Alterne contextos visuais

### 📚 Documentação Anti-alucinação

Cada seção possui documentação MDX detalhada com:

- **Tokens utilizados** com valores exatos
- **Classes Tailwind** importantes
- **Regras críticas** do que pode/não pode ser alterado
- **Exemplos válidos** vs antipadrões
- **Dependências técnicas** necessárias

### 🧪 Acessibilidade

- **Addon A11y** habilitado para auditoria
- **Keyboard navigation** testada
- **Color contrast** validado
- **Screen reader** compatibility

## 🚀 Como Usar

### Para Desenvolvedores

1. **Explore as seções** para entender os padrões
2. **Leia a documentação** antes de implementar
3. **Use os tokens** definidos no design system
4. **Respeite as regras** de cada componente

### Para Designers

1. **Reference visual** para consistency
2. **Tokens precisos** para handoff
3. **Estados documentados** (hover, focus, etc.)
4. **Breakpoints exatos** para responsive

### Para QA

1. **Visual regression** baseline
2. **Acessibilidade** checklist
3. **Responsive** testing
4. **Cross-browser** validation

## ⚠️ Regras Importantes

### ✅ SEMPRE

- Usar tokens de cor definidos
- Manter estrutura de componentes
- Aplicar hover effects padronizados
- Respeitar hierarchy tipográfica
- Testar acessibilidade

### ❌ NUNCA

- Inventar cores fora da paleta
- Quebrar responsividade
- Alterar fonts (Inter/Jost apenas)
- Remover animações sem fallback
- Ignorar documentação das seções
